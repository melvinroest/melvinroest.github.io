Syntax - Config

In the config file several configuration settings can be defined.

config.xml

						<?xml version="1.0" encoding="utf-8"?>
						<ximpel>
							<config>
								<!-- Settings are defined here -->
								<enableControls>true</enableControls>
								<controlsDisplayMethod>overlay</controlsDisplayMethod>
							</config>	
						</ximpel>

The following configuration options are available:

Setting Description Default
enableControls Specifies if XIMPEL will show control buttons (play/pause/etc). Should be true or false. true
controlsDisplayMethod Specifies how XIMPEL will show the controls. Value overlay will show the controls by placing the control bar over the content of the presentation. Value fixed will show the controls below the content (this may cause black bars to be shown around the presentation). overlay
mediaDirectory If a mediaDirectory is specified then all core media types (picture/audio/video) will search for the source of the media items relative to the specified mediaDirectory. Custom media types may also make use of this mediaDirectory setting, but that is up to the media type implementation. Note that the specified mediaDirectory is relative to the html page that includes XIMPEL. No mediaDirectory is used
showScore Specifies if XIMPEL will show the score display on-screen. Should be true or false. false

Overwriting Config file settings

Config settings specified in the config file can be overwritten in the playlist file. So if you have this config file:

config.xml

						<?xml version="1.0" encoding="utf-8"?>
						<ximpel>
							<config>
								<enableControls>true</enableControls>
							</config>	
						</ximpel>

And this playlist file:

playlist.xml

						<?xml version="1.0" encoding="utf-8"?>
						<ximpel>
							<playlist>
								<!-- Description of the presentation comes here -->
							</playlist>
							<config>
								<enableControls>false</enableControls>
							</config>	
						</ximpel>

Then the enableControls setting will get value false, because settings in the config file are overwritten by settings in the playlist file. This is useful if you have a common config file used for all your XIMPEL presentations and want to overwite some settings specifically for a certain playlist.
