Syntax - Filler mediatype

The filler media type does nothing. In some situations it is exactly what you need. See for example here where parallel play, sequential play are mixed with fillers in order to get the timing right.

playlist.xml

						<ximpel>
							<playlist>
							    <subject>
							        <media>
										<filler />
							        </media>
							    </subject>
							</playlist>
						</ximpel>

Infinite fillers vs fillers that have a time duration

If you want a filler that shows infinitely long, then you do not need to give a duration. However, if you want a filler to last for a couple of seconds, then the duration attribute needs to be given like so: 

						<filler duration="10" />

Specific things to know with fillers and parallel play

Also, in order to let the final sequential item disappear a filler needs to be added, like so: 

						<ximpel>
							<playlist>
							<subject>
								<sequence>
									<some_media_item duration="30" />
									<!-- without the following filler, the previous media item will not be removed from screen -->
									<filler duration="1" /> 
									<parallel>
										...
									</parallel>
								</sequence>
							</subject>
							</playlist>
						</ximpel>

This is not an optimal solution. Preferably XIMPEL would work in a way that this filler would not be needed. For now, this is the workaround.
