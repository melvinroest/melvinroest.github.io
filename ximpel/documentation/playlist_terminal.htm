Syntax - Terminal mediatype

This is an advanced media type. It allows for command-line tutorials. Knowledge about clients, servers and a tiny bit of the code of the media type terminal.js is required. The code you need to know is regarding terminal.js is explained on this page.

playlist.xml

						<ximpel>
							<playlist>
							    <subject>
							        <media>
										<terminal />
							        </media>
							    </subject>
							</playlist>
						</ximpel>

The duration attribute

If you want a terminal to last for a certain amount of time, then the duration attribute needs to be given like so: 

						<terminal duration="35" />

This specific terminal will show up for 35 seconds on the screen.

Server setup of the terminal media type

The terminal media type requires a server. The server will handle the terminal commands and execute them remotely. This server could be any type of code able to handle these type of requests. A prototype server has been made at the following Github repository (of Melvin Roest). Of this particular server the standard port is 8080 and is called with node server.js <port_number>.

If you have a custom port number on the NodeJs server (or your own server), then you need to change the port number in the media type terminal.js. This is found on the line socket = io('http://localhost:8080');. Change 8080 to any port number you desire.

Positioning

It is possible to position the terminal media type like any other media type with x, y, width and height.
